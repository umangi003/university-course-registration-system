📘 README – University Course Registration System
📌 Project Overview

This project is a University Course Registration System designed to replace the traditional manual registration process that relies on physical forms and email communication. The system aims to improve efficiency, reduce errors, and provide real-time visibility for students and academic staff during the registration process.

🎯 Objective

The main objective of this system is to:

Digitize the course/module registration process
Reduce manual workload on university staff
Minimize errors in module allocation
Provide students with real-time registration status updates
Improve transparency and processing speed
👥 Key Stakeholders
Students – submit and track module registrations
Registration Staff – review and process requests
Academic Manager – final approval authority
System Administrator – manages system configuration
⚙️ Key Features
Online module registration submission
Automatic module availability checking
Approval workflow (Staff → Manager)
Real-time registration status updates
Audit logging of registration activities
Role-based access for different users
📄 Project Deliverables

This project includes:

Problem Statement analysis
Stakeholder & RACI Matrix
AS-IS and TO-BE process models (BPMN)
Software Requirements Specification (SRS)
Requirements Traceability Matrix (RTM)
Business Case with ROI analysis
💡 Expected Benefits
Faster registration processing time
Reduced administrative workload
Improved accuracy and fewer manual errors
Better student experience through transparency
Higher efficiency during peak registration periods
🛠️ Technologies (Conceptual)
